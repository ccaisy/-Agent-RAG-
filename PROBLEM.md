# 项目问题定义

- **当前状态 (现状)**：本地只有一个初步的项目构想（例如：大模型智能 Agent 或特定 RAG 模块），无基础代码，尚未打通核心链路。
- **目标状态 (目标)**：完成一个可运行、具备核心亮点、能写入简历直接向面试官展示的大模型算法求职项目。
- **状态差距 (差距)**：缺少整体架构设计、模块化的代码实现以及核心算法的调优验证。
- **判断解决的标准 (标准)**：输入测试数据后，系统能够按预期输出结果，且终端无报错退出。

## 求解边界与对象

- **目标约束**：优先实现“核心可用”的 MVP（最小可行性产品），暂不追求极致的性能优化。
- **环境约束**：macOS 系统，仅使用本地 CPU/统一内存或外部 API，Python 开发环境。
- **处理对象**：LLM API 接口调用、数据清洗脚本、核心逻辑串联流。
